# 验证场景

用于手动验证 finance-anomaly skill 是否按预期工作。

---

## 场景 1：正常单据（全维度通过）

**输入（粘贴给 Claude）：**
```json
{
  "amount": 180,
  "date": "2026-03-25",
  "merchant": "麦当劳（中关村店）",
  "expense_type": "餐饮",
  "invoice_no": "04400012345"
}
```

**预期输出：**
- 综合风险等级：🟢 正常
- 异常明细：无异常
- 建议动作：正常通过

**验证点：** 所有维度均无命中，输出简洁，不输出虚假异常。

---

## 场景 2：高风险（拆单 + 连续发票号）

**输入（粘贴给 Claude）：**
```
以下是两张同日报销单，请检查是否有异常：
单据1：金额¥490，2026-03-20，全聚德（王府井），餐饮，发票号00045231
单据2：金额¥480，2026-03-20，全聚德（前门店），餐饮，发票号00045232
```

**预期输出：**
- 综合风险等级：🔴 高风险
- 命中 [频次] 拆单（两张单合计 ¥970，超审批阈值 ¥500，同天同商户名前 4 字相同）
- 命中 [发票] 连续发票号（00045231、00045232）

**验证点：** 正确识别跨两张单据的拆单行为；连续发票号被标记。

---

## 场景 3：非结构化输入追问验证

**输入（粘贴给 Claude）：**
```
帮我看看这张出差单
```

**预期行为：**
Claude 不立即给出报告，而是回复类似：
> "我需要以下信息才能完成检测：金额、消费日期、商户、费用类型，请补充。"

**验证点：** 缺失 4 个必须字段时正确触发追问，不产生空报告。

---

## 场景 4：字段部分缺失（优雅降级）

**输入（粘贴给 Claude）：**
```json
{
  "amount": 300,
  "date": "2026-03-22",
  "expense_type": "餐饮",
  "merchant": "外婆家"
}
```

**预期输出：**
- 可检测维度（金额、时间、合规）正常输出结果
- 报告末尾注明：
  - "[频次] 因信息不足未作检测"（无 recent_records）
  - "[发票] 因信息不足未作检测"（无 invoice_no）

**验证点：** 缺失维度被跳过且末尾注明，不报错，不留空白行。

---

## 场景 5：合规超标（差旅标准）

**输入（粘贴给 Claude）：**
```json
{
  "amount": 1500,
  "date": "2026-03-18",
  "merchant": "上海四季酒店",
  "expense_type": "住宿",
  "city": "上海"
}
```

**预期输出：**
- 综合风险等级：🔴 高风险
- 命中 [合规] 上海住宿 ¥1,500/晚，超出北上广深标准（¥1,200/晚）25%

**验证点：** 城市匹配正确（上海 → 北上广深标准），风险等级正确。
