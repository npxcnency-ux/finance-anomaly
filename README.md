# finance-anomaly

财务费用异常识别 Skill —— 用于 Claude Code 的报销单据异常检测规则引擎。

## 功能

对报销单据进行五维度并行检测，输出结构化风险报告：

| 维度 | 检测内容 |
|------|---------|
| 金额 | 超标准上限、整数金额疑点、超申请人历史均值 |
| 时间 | 非工作日/法定假日消费、深夜消费、发票日期与消费日期不符 |
| 频次 | 同商户重复消费、拆单规避审批、相同金额重复出现 |
| 合规 | 差旅标准超标、费用类型与商户不匹配、超单次报销上限 |
| 发票 | 重复发票号、发票号格式异常、连续发票号 |

风险等级分三档：🔴 高风险（退回/升级审批）、🟡 中风险（人工复核）、🟢 正常（通过）。

## 使用方式

### 自动节点模式

审批系统传入结构化 JSON，直接输出报告：

```json
{
  "amount": 1500,
  "date": "2026-03-29",
  "merchant": "全聚德（王府井店）",
  "expense_type": "餐饮",
  "invoice_no": "04412226",
  "is_duplicate_invoice": true
}
```

### 交互式模式

财务人员用自然语言描述：

> "帮我看看这个单子有没有问题：小王报了一张1200元的住宿，说是去上海出差，3月28号的"

Skill 自动提取字段，缺失必要信息时追问一次。

## 输入字段

| 字段 | 说明 | 必须 |
|-----|------|:----:|
| amount | 报销金额（人民币） | 是 |
| date | 消费日期（YYYY-MM-DD） | 是 |
| merchant | 商户名称 | 是 |
| expense_type | 费用类型（餐饮/差旅/办公用品/招待等） | 是 |
| invoice_no | 发票号码 | 否 |
| time | 消费时间（HH:MM） | 否 |
| city | 出行城市 | 否 |
| submitter_history | 申请人历史同类消费均值 | 否 |
| recent_records | 申请人近 30 天报销记录列表 | 否 |
| is_duplicate_invoice | 系统比对结果（true/false） | 否 |

可选字段缺失时对应维度静默跳过，在报告末尾注明未检测原因。

## 输出示例

```
## 费用异常分析报告

**单据信息：** ¥1,500 | 2026-03-29 | 全聚德（王府井店）| 餐饮

**综合风险等级：** 🔴 高风险

**异常明细：**
- [金额] 餐饮费 ¥1,500，超出标准上限（¥200/人）明显，高风险
- [时间] 消费发生在周日（2026-03-29），非工作日，中风险
- [发票] 发票号 04412226 已存在重复报销记录，高风险

**建议动作：** 退回补充材料
**理由：** 发票重复且金额严重超标，需申请人提供原件核查
```

## 评估与验证

- **自动化评估**：`evals/evals.json` 包含 4 个测试用例，每个用例定义输入 prompt 和 expectations 数组
- **手动验证**：`tests/validation-scenarios.md` 包含 5 个场景，覆盖正常单据、高风险拆单、追问流程、优雅降级、合规超标
