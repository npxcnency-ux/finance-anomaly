# 财务费用异常识别 Skill

## 触发场景

**自动节点模式：** 审批系统将结构化单据字段传入，直接运行检测并输出报告。
**交互式模式：** 财务人员粘贴数据、描述问题或提问，灵活运行检测。

---

## 第一步：识别触发模式与输入格式

### 判断触发模式

| 特征 | 模式 |
|-----|------|
| 输入包含明确的 JSON 结构或系统字段标签 | 自动节点模式 |
| 输入为自然语言、表格粘贴或混合文本 | 交互式模式 |

### 判断输入格式

**结构化输入（直接映射）：**
输入包含明确分隔的字段（JSON、表格、键值对），直接提取以下字段：

| 字段名 | 说明 | 是否必须 |
|-------|------|---------|
| amount | 报销金额（数字，人民币） | 必须 |
| date | 消费日期（YYYY-MM-DD） | 必须 |
| merchant | 商户名称 | 必须 |
| expense_type | 费用类型（餐饮/差旅/办公用品/招待等） | 必须 |
| invoice_no | 发票号码 | 可选 |
| time | 消费时间（HH:MM） | 可选 |
| city | 出行城市 | 可选 |
| submitter_history | 申请人历史同类消费均值 | 可选 |
| recent_records | 申请人近 30 天报销记录列表 | 可选 |
| is_duplicate_invoice | 系统比对结果（true/false） | 可选 |

**非结构化输入（提取+追问）：**
从自然语言中尽可能提取上述字段。提取完成后，若 amount、date、merchant、expense_type 任一缺失，在检测前向用户追问：

> "我需要以下信息才能完成检测：[缺失字段列表]，请补充。"

追问一次即可，不重复追问。获得补充后继续检测。

---

## 第二步：并行运行五维度检测

字段就绪后，并行加载并执行以下规则文件，各维度独立判断：

1. `rules/amount.md` — 金额异常检测（需要：amount, expense_type, submitter_history）
2. `rules/time.md` — 时间异常检测（需要：date, time, expense_type）
3. `rules/frequency.md` — 频次异常检测（需要：merchant, date, amount, recent_records）
4. `rules/compliance.md` — 合规检查（需要：amount, expense_type, merchant, city）
5. `rules/invoice.md` — 发票检测（需要：invoice_no, is_duplicate_invoice, date, merchant）

**字段缺失处理：** 每个维度的规则文件内部已标注"无XX数据时跳过"。遵循各规则文件的跳过规则；无法运行的维度在报告末尾注明"[维度名] 因信息不足未作检测"，不输出错误。

---

## 第三步：汇总结果，生成报告

1. 收集所有命中的异常项
2. 综合风险等级 = 所有命中项中最高的风险等级
3. 无任何命中项 → 综合风险等级 = 正常
4. 按 `output-template.md` 中的格式生成报告：
   - 自动节点模式 → 使用"自动节点模式"格式
   - 交互式模式 → 使用"交互式模式"格式

---

## 示例

### 示例 1：结构化输入（自动节点模式）

**输入：**
```json
{
  "amount": 1500,
  "date": "2026-03-29",
  "merchant": "全聚德（王府井店）",
  "expense_type": "餐饮",
  "invoice_no": "04412226",
  "is_duplicate_invoice": true
}
```

**输出：**
```
## 费用异常分析报告

**单据信息：** ¥1,500 | 2026-03-29 | 全聚德（王府井店）| 餐饮

**综合风险等级：** 🔴 高风险

**异常明细：**
- [金额] 餐饮费 ¥1,500，超出标准上限（¥200/人）明显，高风险
- [时间] 消费发生在周日（2026-03-29），非工作日，中风险
- [发票] 发票号 04412226 已存在重复报销记录，高风险

**建议动作：** 退回补充材料
**理由：** 发票重复且金额严重超标，需申请人提供原件核查
```

### 示例 2：非结构化输入（交互式模式）

**输入：** "帮我看看这个单子有没有问题：小王报了一张1200元的住宿，说是去上海出差，3月28号的"

**处理过程：** 提取字段 → amount=1200, date=2026-03-28, expense_type=住宿, city=上海；merchant 和 invoice_no 均未提供。merchant 为必须字段，追问用户后未补充，以"未知商户"继续检测（频次维度和发票维度跳过）。

**输出：**
```
## 费用异常分析报告

**单据信息：** ¥1,200 | 2026-03-28 | 未提供商户 | 住宿（上海出差）

**综合风险等级：** 🟡 中风险

**异常明细：**
- [合规] 上海住宿 ¥1,200/晚，超出北上广深标准（¥800/晚）50%，高风险

**建议动作：** 人工复核
**理由：** 住宿费超出差旅标准，建议核实酒店收据或要求提供出差审批单

[频次] 因未提供商户名称，频次异常未作检测。
[发票] 因未提供发票号，发票真伪未作检测。如需核查，请补充发票号。
```
